Техническое задание: Gemini CLI — Финансовый AI-бот
Цель проекта

Создать телеграм-бота, который:

Получает тикер и период от пользователя.

Загружает исторические данные (Tradernet/yfinance).

Строит график цены с уровнями поддержки и сопротивления.

Анализирует данные через OpenAI и дает прогноз и комментарий.

Хранит историю запросов и графиков в базе данных.

Поддерживает интерактивный UI и асинхронную обработку.

1️⃣ UI / UX
Требования

Использовать aiogram 3 с FSMContext для состояний.

Переходы:

/start → ввод тикера → выбор периода → получение анализа.

Inline-кнопки:

📊 «График с индикаторами»

📈 «Прогноз»

🔁 «Новый тикер»

🕘 «История»

Графики:

PNG через matplotlib или интерактивные Plotly/HTML (опционально).

Приветствие и инструкции в ответах.

Пример диалога
Пользователь: /start
Бот: Привет! Отправь тикер.
Пользователь: AAPL
Бот: Выберите период анализа:
[1д] [1м] [6м] [1г] [всё]
Пользователь: 1м
Бот: Запрашиваю данные по AAPL за 1 месяц...
Бот: Анализ готов! 📊
       Тренд: восходящий
       Прогноз: рост
       Комментарий: Покупка возможна
[График с индикаторами] [Прогноз] [Новый тикер] [История]

2️⃣ AI / Анализ

Используем OpenAI GPT-5 (или GPT-4 при необходимости).

Входные данные:

Последние 100–200 строк исторических данных: open, high, low, close, volume.

Период прогноза: 1д, 1м, 6м, 1г.

Выходные данные — строго JSON:

{
  "trend": "восходящий",
  "support": [145.3, 148.1],
  "resistance": [152.0, 155.5],
  "forecast": "Цена вероятно пойдет вверх к сопротивлению 152",
  "comment": "Возможна покупка при пробое 148"
}


Детальный промпт для AI:

Ты опытный финансовый аналитик. Анализируй данные акции {ticker} (open, high, low, close, volume).
Сделай:
1. Определи текущий тренд: восходящий/нисходящий/флэт
2. Определи ключевые уровни поддержки и сопротивления
3. Сделай прогноз движения цены на период {horizon}
4. Дай краткий инвестиционный комментарий

Ответ строго в JSON формате с полями: trend, support, resistance, forecast, comment.


Функции AI:

analyze_with_openai(df, ticker, horizon) → JSON

Возможность обработки ошибок, если JSON невалидный.

3️⃣ База данных

Используем SQLite (легкая и встроенная). В будущем можно перейти на PostgreSQL.

Таблицы:

CREATE TABLE users (
    id INTEGER PRIMARY KEY,
    username TEXT,
    first_name TEXT,
    last_name TEXT
);

CREATE TABLE requests (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER,
    ticker TEXT,
    period TEXT,
    request_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    trend TEXT,
    forecast TEXT,
    comment TEXT,
    FOREIGN KEY(user_id) REFERENCES users(id)
);

CREATE TABLE graphs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    request_id INTEGER,
    filepath TEXT,
    FOREIGN KEY(request_id) REFERENCES requests(id)
);


Функции:

Сохранять каждый запрос и анализ.

Кэшировать графики для повторного использования.

Возможность просмотра истории через команду /history.

4️⃣ Асинхронность и обработка данных

Асинхронный телеграм-бот с aiogram:

Все хендлеры async def.

Длительные операции (OpenAI API, построение графика) через asyncio.create_task.

Пример:

await message.reply("Анализ в процессе, подождите...")
asyncio.create_task(process_ticker(user_id, ticker, period, message.chat.id))


Кэширование:

Если тикер + период уже есть в базе → отдавать график и анализ из БД, не запрашивая API.

5️⃣ Новые функции / улучшения

Уведомления: бот может слать alert, если цена пересекла уровень поддержки/сопротивления.

История пользователя: команда /history → список всех предыдущих анализов с кнопками для просмотра графиков.

Дополнительные индикаторы:

Цветовые обозначения на графике.

Интерактивный график (опционально) через Plotly.
